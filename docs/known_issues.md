| Some of the tutorials in VectorBlox SDK uses the open source tool [**onnx2tf**](https://github.com/PINTO0309/onnx2tf) to convert an ONNX file to a TFLite file. Recently the **onnx2tf** repository made [changes](https://github.com/PINTO0309/onnx2tf/releases/tag/2.3.17) and removed a calibration file. This breaks the older versions of the **onnx2tf** tool, and as a result the VectorBlox tutorials using **onnx2tf** fails.<br> **Workaround** <br> Download the file [calibration_image_sample_data_20x128x128x3_float32.npy](https://github.com/Microchip-Vectorblox/assets/raw/refs/heads/main/npy_files/calibration_image_sample_data_20x128x128x3_float32.npy) in the tutorial directory before running **onnx2tf**. You can use the following command to download the file from command line:<br> `wget --no-check-certificate https://github.com/Microchip-Vectorblox/assets/raw/refs/heads/main/npy_files/calibration_image_sample_data_20x128x128x3_float32.npy` <br> **NOTE** User will face the same issue if **onnx2tf** is used to convert custom models to TFLite format. User should download the calibration_image_sample_data_20x128x128x3_float32.npy file before running **onnx2tf** tool.| All |  | Open |
